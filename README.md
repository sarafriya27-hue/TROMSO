# Insurance Claim Settlement — Bias & Predictive Analysis

A Streamlit dashboard + supporting scripts to investigate potential bias in
insurance claim settlement decisions and to build classification models
that predict claim outcomes (Approved vs Repudiated).

## What's inside

```
insurance_project/
├── app.py                 # Streamlit dashboard (main deliverable)
├── run_analysis.py        # Standalone script — same analysis, no Streamlit needed
├── utils.py                # Shared data cleaning / feature engineering / ML code
├── requirements.txt
├── data/
│   └── Insurance.csv       # Your dataset (bundled sample)
└── outputs/                 # Generated by run_analysis.py (tables, plots, findings.txt)
```

## 1. Run locally

```bash
cd insurance_project
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

# Option A: Interactive dashboard
streamlit run app.py

# Option B: Static / scripted run (no browser needed)
python run_analysis.py --input data/Insurance.csv --outdir outputs
```

The dashboard opens at `http://localhost:8501`. Use the sidebar to either use
the bundled `data/Insurance.csv` or upload your own claims CSV (same column
structure).

## 2. Deploy on Streamlit Community Cloud (free)

1. Create a new GitHub repository and push everything in this folder
   (including `data/Insurance.csv` if you're OK sharing it, otherwise users
   can upload their own file via the sidebar).
   ```bash
   git init
   git add .
   git commit -m "Initial commit - claim settlement bias dashboard"
   git branch -M main
   git remote add origin https://github.com/<your-username>/<your-repo>.git
   git push -u origin main
   ```
2. Go to **https://share.streamlit.io** → "New app".
3. Select your repo, branch `main`, and set the main file path to `app.py`.
4. Click **Deploy**. Streamlit Cloud will install `requirements.txt`
   automatically and host the app at a public URL you can share with your
   team/manager.

## 3. Dataset columns expected

|---|---|
| POLICY_NO | Policy identifier |
| PI_GENDER | Policyholder gender |
| SUM_ASSURED | Sum assured (₹) |
| ZONE | Underwriting team / zone |
| PAYMENT_MODE | Premium payment frequency |
| EARLY_NON | Whether claim was an "early" claim |
| PI_OCCUPATION | Policyholder occupation |
| MEDICAL_NONMED | Whether a medical exam was done |
| PI_STATE | Policyholder's state |
| REASON_FOR_CLAIM | Cause of death/claim |
| PI_AGE | Policyholder age |
| PI_ANNUAL_INCOME | Policyholder annual income (₹) |
| POLICY_STATUS | **Target** — Approved Death Claim / Repudiate Death |

## 4. What each dashboard tab does

- **Overview** — high-level KPIs and distribution charts.
- **Cross-Tabulation** — Objective 1: cross-tabs of any dimension (age,
  income, team, gender, occupation, etc.) against policy status, with
  counts, row-percentages, and approval-rate bar charts.
- **Bias Diagnostics** — Objective 2: chi-square independence tests across
  9 dimensions, approval-rate tables/charts per dimension, and two
  multi-dimensional heatmaps (Age×Income, Team×Gender).
- **Predictive Modeling** — Objectives 3 & 4: feature engineering pipeline
  (scaling + one-hot encoding + imputation) feeding KNN, Decision Tree,
  Random Forest, and Gradient Boosting classifiers; metrics table
  (train/test accuracy, precision, recall, F1, ROC-AUC), train-vs-test
  accuracy chart, ROC curves, confusion matrices, and tree feature
  importance charts.
- **Findings & Recommendations** — Objective 5: auto-generated summary of
  statistically significant bias dimensions, approval-rate gaps flagged
  for review, and concrete recommendations.

## Notes & caveats

Statistical association (chi-square significance, approval-rate gaps)
indicates the settlement outcome is *not independent* of a given factor —
it is a **diagnostic signal**, not proof of intentional discrimination.
Treat findings as a starting point for a deeper actuarial/compliance review,
not a final verdict.
